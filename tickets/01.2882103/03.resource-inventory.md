## Confidence Model

Every resource entry carries a confidence label. This is a migration project with a previous failed attempt — being explicit about what is known vs assumed is not a weakness, it is the only honest way to operate.

| Label | Meaning |
|---|---|
| `verified` | Directly visible and named in the ARB diagram — treated as source of truth |
| `assumed` | Inferred from diagram shape, connection, or pattern — needs team confirmation |
| `?` | Unknown or ambiguous — either my knowledge gap (new to project) or a knowledge bottleneck from core team — must be explicitly resolved before implementation |

> Knowledge bottlenecks are two-sided: the new team member who does not yet know, and the core member whose knowledge is not yet written down. Both are risks. Both should produce a `?` entry rather than silence.

## Resource Naming Pattern

```
prnd-promo-<service>-<resource-descriptor>
prmd-promo-<datastore-type>-<collection-descriptor>
```

- `prnd` — promo next-gen deployment prefix
- `prmd` — promo data prefix
- `run` — Cloud Run
- `cron` — Cloud Scheduler
- `fs` — Firestore
- `bq` — BigQuery
- `sm` — Secret Manager
- `gcp` — Cloud Storage bucket
